Terminology

Definition: When we say that a metric $d_M$ is stable, we imply that $d_M(f,g) \leq ||f-g||_{\infty}$. In this case, $f$ and $g$ are the functions corresponding to two scalar fields $(\mathbb{X},f)$ and $(\mathbb{X},g)$. Note that in this case, it only makes sense to have $f$ and $g$ defined on the same domain. Furthermore, $f$ and $g$ in $d_M$ are assumed to be "encoded properly". By this, we mean that if the distance measure $M$ is defined on Reeb graphs, then the input to $d_M$ are the Reeb graphs corresponding to the functions $f,g$. This might help us decode statements like $d_I(f,g) \geq d_B(f,g)$, where $I$ refers to the interleaving distance of Reeb graphs and $B$ refers to the bottleneck distance between persistence diagrams.

While this ambiguous notation might help keep things short in some situations, it is important to know that the interleaving distance has varying definitions depending on Reeb Graphs/Persistence modules, and Bottleneck Distance also has varying degrees as well

Definition: We say that a metric is $d_M$ is discriminative if $d_M(f,g) \geq d_B(f,g)$, where $B$ is the bottleneck distance between persistence diagrams.

Questions

In terms of density of this document, how much should we dive into the definitions of interleaving, functional distortion, and edit distance? My personal opinion is to put as much information for the interleaving distance that allows a reader to use this document as a source to actually understand how the distance works. In this case, we'd need to define some information from category theory and then go into pre-cosheafs. I think this can be done in a reasonably sized piece (ignoring constructibility).

The other idea is to defer the reader to more intense information, but I think that kind of defeats the purpose of the document. If we're going to have examples for each of the distances, the definitions should be pretty rigorous, IMO

In general, I think it is wise to make sure this document can serve as a central location for definitions of these structures since the goal is to compare them so heavily.

Maybe I'm thinking of this survey too broadly, though. To me, it almost feels like it would be a "manual" or like the a section in a textbook. I'm not sure if that's good or bad.
